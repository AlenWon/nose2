Outputting XML Test Reports¶

Note

New in version 0.2

Output test reports in junit-xml format.

This plugin implements startTest(), testOutcome() and stopTestRun() to compile and then output a test report in junit-xml format. By default, the report is written to a file called nose2-junit.xml in the current working directory. You can configure the output filename by setting path in a [junit-xml] section in a config file.

Configuration [junit-xml]¶

always-on¶
Default :False
Type :boolean
path¶
Default :nose2-junit.xml
Type :str

Sample configuration¶

The default configuration is equivalent to including the following in a unittest.cfg file.

[junit-xml]
always-on = False
path = nose2-junit.xml

Command-line options¶

-X DEFAULT, --junit-xml DEFAULT¶

Generate junit-xml output report
